1) Requirements

  • Cyberpunk 2077 (PC)
  • WolvenKit (latest recommended)
  • Blender
  • (Optional but common) Cyber Engine Tweaks (CET) for console spawning
This guide focuses on the WolvenKit finishing pipeline: importing, building, and installing the mod.

2) Blender Preparation (Brief)

You already did most of this. The only key output WolvenKit expects is a .GLB export that matches the game mesh structure.

  • Import Sketchfab shoe (FBX/GLB)
  • Align to the right foot, apply transforms (Ctrl + A → Apply All)
  • Weight the shoe fully to RightFoot (or the correct foot bone)
  • Export as GLB (recommended for WolvenKit import)
If you exported FBX earlier: that’s fine for Blender work, but WolvenKit’s mesh import expects .GLB for import/export workflows.

3) WolvenKit Core Steps (Main Part)

Step A — Create a Clean Mod Project

  • Open WolvenKit → New Project
  • Name it (example: HighHeelsWearable)
  • Confirm the game path / version

Step B — Pick a Base Shoe Mesh to Replace (Fastest Path)

The easiest first success is a replacer (replace an existing shoe in-game).

  • In WolvenKit, browse the game files for an existing footwear mesh (.mesh)
  • Export that mesh to GLB so you have the correct structure/rig baseline
  • Use that exported GLB as the base you import your custom shoe onto
This matches WolvenKit’s rule: you import your .GLB into an existing .mesh.

Step C — Import Your Custom GLB Over the Base .mesh

  1. Open Tools → Import/Export Tool
  2. Set Target File Format to Mesh
  3. Select the base shoe .mesh you want to overwrite
  4. Select your exported custom .GLB
  5. Run Import
Success looks like: WolvenKit completes import without fatal errors, and your project now contains the edited .mesh (and related files).

Step D — Build the Mod

  • Click Build in WolvenKit
  • WolvenKit outputs archives for your mod
Want a “new item” instead of replacer? That’s a longer pipeline (ArchiveXL/TweakXL). Use the “Adding new items” guide in the links section.

4) Install & Test In-Game

Where to put the mod

Place the built archive(s) into:

Cyberpunk 2077\archive\pc\mod

Testing

  • Launch the game
  • Equip the base shoe item you replaced (replacer workflow)

(Optional) Spawn / Debug via CET

If you’re doing a “new item” workflow, CET is commonly used to spawn items via console.

See the verified CET install/debug videos in the links section.

5) Troubleshooting (Quick Fixes)

Import/Export Tool buttons are greyed out

  • Make sure you are inside a mod project (not just browsing game archives)
  • Try closing and reopening WolvenKit
  • Open the tool from the menu: Tools → Import/Export Tool

“Bone Heat Weighting failed” in Blender

  • For shoes, prefer: Empty Groups + assign all vertices weight 1.0 to the correct foot bone

Foot clipping through the shoe

  • Don’t over-scale the shoe too much
  • Common fix: delete the foot faces that are fully inside the shoe (edit mode)

Import errors in WolvenKit

WolvenKit expects .GLB and you must import over an existing .mesh. If you try to “convert FBX to mesh” directly, it usually won’t work in the standard pipeline.
